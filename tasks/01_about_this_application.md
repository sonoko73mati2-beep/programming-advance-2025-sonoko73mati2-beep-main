# このアプリケーションについて

芸術専門学群の「プログラミング応用」で、開発の雛形となる基礎的な設定が行われたアプリケーションです。
以下の仕様で設計します。

1. Webブラウザで動作するアプリケーションであること
2. Javascript で開発されていること
3. Javascript は ES6に準拠していること
4. ゲームエンジンとしてPixiJS 7.0を用いること。
5. ゲームのルートから配置する要素(各画面やゲームキャラクターなど)はクラスとしてファイルを分割してプログラミングすること
6. 画面ウィンドウサイズに対して最大までCanvasを表示し、ウィンドウサイズの変更に追従すること
7. スマートフォン上のブラウザで動作することを考慮した設計であること
8. マルチタッチイベントに対応していること
9. ファイル構成としてはルートディレクトリ以下に以下の構造で配置すること
   1. index.html - エントリーポイントとなるHTMLファイル
   2. src/ - JavaScriptソースコードが格納されたディレクトリ
      - lib/ - 外部ライブラリ（PixiJS等）を格納
      - character.js - キャラクタークラス（PIXI.Containerを継承、タッチイベント処理を担当）
      - game.js - ゲームのステージクラス（PIXI.Containerを継承、キャラクターを配置・管理）
      - index.js - アプリケーションの初期化とエントリーポイント（Initクラスを含む）
   3. css/ - index.htmlが使用するCSSが格納されたディレクトリ
   4. images/ - ロードする画像ファイルが格納されたディレクトリ
   5. sounds/ - ロードする音声ファイルが格納されたディレクトリ
   6. tasks/ - プロジェクトに関するドキュメントが格納されたディレクトリ

## 初期設定のクラス構成

### Init（index.js内）
- PixiJSアプリケーションの初期化を担当
- Canvas要素の生成とDOMへの追加
- ウィンドウリサイズへの対応

### Game
- ゲームのステージとして機能するコンテナクラス
- 背景の描画とリサイズ対応
- キャラクターの配置と管理
- 毎フレームのupdate処理

### Character
- タップ/クリック可能なキャラクターオブジェクト
- PIXI.Graphicsで円形を描画
- pointerdown/pointerupイベントに対応
- hitAreaによるタッチ判定領域の設定
